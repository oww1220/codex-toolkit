# DESIGN-BRIEF — content-platform (발췌)

> `/design-brief` 산출. 제품·사용자 인터뷰를 정리한 문서. 아래는 핵심 발췌다.
> (전체 산출물에는 화면 인벤토리·경쟁 서비스·제약 조건 등이 더 포함된다.)

## 1. 제품 정의

- **무엇**: 긴 글·에세이·리포트를 큐레이션해 읽히는 콘텐츠 서비스.
- **핵심 가치**: "읽는 경험" 자체. 글이 주인공, UI는 조연.
- **플랫폼**: 웹 우선(데스크톱·모바일 반응형). 모바일 독서 비중 높음.
- **규모**: 글 수백~수천 편, 분류 10여 개, 동시 운영자 1인 + 외부 필자.

## 2. 사용자 · JTBD

| 사용자 | 핵심 작업(Job) | 빈도 |
|--------|----------------|------|
| 독자 | 읽을 만한 글을 **발견**하고 **끝까지 읽는다** | 매일 |
| 독자 | 분류/태그로 **비슷한 글을 더 찾는다** | 자주 |
| 독자 | 키워드로 **검색**한다 | 가끔 |
| 독자 | 마음에 든 글을 **저장·공유**한다 | 자주 |
| 운영자 | 글을 발행하고 분류·추천을 정리한다 | 매일 |

핵심 동선: **홈(발견) → 목록(분류) → 상세(읽기) → 저장/공유** + 검색은 어디서나.

## 3. 인터뷰에서 끌어낸 취향 신호

원문(사용자 말) → 디자인 신호로 메모만 해 둔다. 번역은 `/design-translate` 단계가 한다.

| 사용자 발언 | 끌어낸 신호 |
|-------------|-------------|
| "뉴스레터보다 잡지" | editorial 톤, serif 가능, 위계 강함 |
| "글이 주인공, UI는 뒤로" | 저장식, 단일 읽기 컬럼, 장식 절제 |
| "오래 읽어도 눈이 안 피로" | 본문 가독성 최우선, 넉넉한 행간/여백 |
| "스크롤하다 길을 잃지 않게" | 분류·진행 표시, 무한 그리드 회피 |
| "스타트업 SaaS / 떠다니는 카드 싫음" | 카드 남발 금지, 거대 히어로 금지, 그림자 절제 |
| "올드해 보이는 것도 싫음" | 모던한 serif + 깨끗한 sans 페어링, 클래식 신문 회피 |

## 4. 확정 vs 추정 (분리해서 기록)

- **확정(confirmed)**: 읽기 중심, 단일 컬럼, 모바일 독서 비중 높음, 잡지 톤, 카드 그리드 거부.
- **추정(assumed)**: serif display 채택 가능(사용자 미확정 — 후보로 검증), 강조색 1색 절제, 저장/공유는 본문 하단 + 떠 있는 보조 바.

추정 항목은 `/design-compare` 후보로 만들어 **사용자 선택으로 검증**한다. 가정으로 확정하지 않는다.

## 5. 명시적 금지 (사용자 발화 기반)

- 떠다니는 카드 그리드 / 거대 히어로 카피 / 스타트업 SaaS 대시보드 톤
- 채도 높은 보라·파랑 그라데이션 / 글로우 / 균등 썸네일 타일
- 순수 검정 본문 위 순수 흰 배경(눈부심) — 살짝 톤 띤 paper 배경 선호

## 6. 비교에 쓸 대표 화면 (콘텐츠 유형 기준)

홈(발견) · 목록(분류) · 상세(읽기) · 검색. — 모든 후보가 **같은 4화면·같은 글·같은 문장**으로
비교된다(6원칙 #4: 동일 조건 비교).
