# DESIGN-BRIEF — admin-dashboard (발췌)

> `/design-brief` 산출. 제품·사용자 인터뷰를 정리한 문서. 아래는 핵심 발췌다.
> (전체 산출물에는 화면 인벤토리·권한 모델·데이터 규모 등이 더 포함된다.)

## 1. 제품 정의

- **무엇**: 주문·회원·정산을 운영하는 사내 관리자 도구(admin).
- **핵심 가치**: 처리 속도와 정보 밀도. 같은 작업을 하루 수십~수백 번 반복.
- **플랫폼**: 데스크톱 웹 우선(상시 큰 모니터). 모바일은 조회·승인 정도만.
- **규모**: 행 수만~수십만, 컬럼 10~20개, 동시 운영자 5~30명, 권한 분리.

## 2. 사용자 · JTBD

| 사용자 | 핵심 작업(Job) | 빈도 |
|--------|----------------|------|
| 운영자 | 조건으로 **검색·필터해 좁힌다** | 매우 잦음 |
| 운영자 | 표에서 **빠르게 훑고 정렬한다** | 매우 잦음 |
| 운영자 | 여러 행을 **일괄 선택·처리한다**(승인/반려/태그) | 잦음 |
| 운영자 | 한 건을 **상세 패널에서 빠르게 편집한다** | 잦음 |
| 관리자 | 권한/설정을 관리한다 | 가끔 |

핵심 동선: **검색·필터 → 표 → 일괄 처리 → 상세 편집**. 화면 전환 최소, 키보드 우선.

## 3. 인터뷰에서 끌어낸 취향 신호

| 사용자 발언 | 끌어낸 신호 |
|-------------|-------------|
| "예쁜 것보다 빠른 게 중요" | 속도·효율 우선, 장식 절제 |
| "정보가 최대한 많이 보였으면" | high density, compact 행 높이, 작은 여백 |
| "필터·검색·일괄처리가 빨라야" | 상단 명령형 검색, 일괄 액션 바, 단축키 |
| "엑셀처럼 표가 중심" | table-first, 고정 헤더, 정렬·인라인 편집 |
| "강조색 많으면 헷갈림" | accent 1색 절제, 상태는 semantic 소수 |
| "카드 누르고 다니는 SaaS 싫음" | 카드 그리드 대시보드 거부, 마케팅 톤 회피 |

## 4. 확정 vs 추정 (분리해서 기록)

- **확정(confirmed)**: 테이블 중심, 높은 정보 밀도, 강력한 검색·필터, 일괄 처리, 강조색 절제.
- **추정(assumed)**: 상세는 화면 전환 대신 우측 슬라이드 패널, 키보드 단축키 상시 노출, 행 높이 compact 기본.

추정은 `/design-compare` 후보로 만들어 **사용자 선택으로 검증**한다.

## 5. 명시적 금지 (사용자 발화 기반)

- 카드 그리드 대시보드 / 마케팅 SaaS 랜딩 톤 / 거대 통계 위젯 나열
- 채도 높은 멀티 강조색 / 그라데이션 헤더 / 과도한 둥근 모서리(읽기 어려운 표)
- 가짜 KPI 숫자(99% / +120%) — 실제 데이터 없는 장식 통계 금지

## 6. 비교에 쓸 대표 화면 (admin 유형 기준)

목록(표) · 필터 패널 · 상세 편집 패널 · 일괄 처리. — 모든 후보가 **같은 4화면·같은 데이터·같은
컬럼**으로 비교된다(6원칙 #4: 동일 조건 비교).
